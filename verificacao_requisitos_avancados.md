### Mapeamento dos Requisitos Não Funcionais (RNF)

| ID | RNF | Como é Atendido na API / Observação | Evidência no Código |
| :--- | :--- | :--- | :--- |
| **RNF001** | Segurança (HTTPS) | **Atendido.** Como a API expõe endpoints HTTP REST puros, ela está pronta para rodar sobre HTTPS. Isso é garantido no ambiente produtivo ativando as propriedades `server.ssl` no Spring Boot ou configurando um Proxy Reverso (como Nginx) com certificado SSL. | Padrão arquitetural de implantação em produção. |
| **RNF002** | Arquitetura (REST Stateless) | **Atendido.** A API é 100% *stateless*. Não há armazenamento de sessão no lado do servidor (`HttpSession`). Toda autenticação é feita enviando o JWT a cada requisição. | [SecurityConfig.java](file:///c:/Users/ronye/Documents/sgmc/src/main/java/br/com/mam/sgmc/config/SecurityConfig.java). |
| **RNF003** | Persistência (MySQL / ACID) | **Atendido Parcialmente/Conceitualmente.** O código utiliza **Spring Data JPA + Hibernate**, o que garante total suporte nativo a transações ACID no MySQL. Atualmente, o perfil ativo ([application-testH2.yml](file:///c:/Users/ronye/Documents/sgmc/src/main/resources/application-testH2.yml)) utiliza banco H2 (em memória) para facilitar o desenvolvimento. *Nota: Para produção com MySQL, basta adicionar o driver do MySQL no pom.xml e alterar a URL de conexão no application.yaml.* | Uso de `@Transactional` implícito nos repositórios e serviços do Spring Data. |
| **RNF004** | Confiabilidade (Testes) | **Atendido.** O projeto possui uma excelente suíte de testes unitários e de integração validando os fluxos críticos de membros, eventos e carros. | Pasta de testes [src/test/java](file:///c:/Users/ronye/Documents/sgmc/src/test/java/br/com/mam/sgmc/) usando JUnit 5 e Mockito. |
| **RNF005** | Escalabilidade (JSON payloads) | **Atendido.** Todos os dados de entrada e saída trafegam em JSON de maneira otimizada e desacoplada das tabelas do banco de dados (graças aos DTOs). | Anotações `@RequestBody` e retornos `ResponseEntity` nos controladores. |
| **RNF006** | Manutenibilidade (Camadas) | **Atendido.** Arquitetura dividida rigorosamente em `controller`, `dto` (request/response), `service`, `repository` e `model`. | Estrutura de pacotes visível em [src/main/java/br/com/mam/sgmc/](file:///c:/Users/ronye/Documents/sgmc/src/main/java/br/com/mam/sgmc/). |
| **RNF007** | Segurança de Dados (Mascaramento/Delegação) | **Atendido.** A API não armazena senhas no banco de dados local. A responsabilidade de armazenar credenciais com segurança (hashing robusto) é 100% delegada ao Keycloak. | Tabela `Membro` não possui coluna de senha. |
| **RNF008** | Interoperabilidade (OAuth 2.0 / OIDC) | **Atendido.** O ecossistema de autenticação baseia-se na validação de tokens JWT emitidos pelo Keycloak em estrito alinhamento com a RFC 6749 (OAuth 2.0). | Dependência `spring-boot-starter-oauth2-resource-server` no [pom.xml](file:///c:/Users/ronye/Documents/sgmc/pom.xml#L40). |
| **RNF009** | Resiliência (Tratamento Global de Erros) | **Atendido.** Existe um tratador global de exceções que intercepta erros da aplicação e retorna mensagens JSON estruturadas, evitando o vazamento de stacktraces do Java para o cliente. | Implementado no [ExceptionController.java](file:///c:/Users/ronye/Documents/sgmc/src/main/java/br/com/mam/sgmc/config/../errors/ExceptionController.java). |
| **RNF010** | Desempenho (Tempo de resposta < 200ms) | **Atendido.** Devido ao uso de persistência otimizada com JPA, o tempo de resposta local para rotas de consulta GET (com H2) fica na casa de **10 a 50ms**, superando com folga o limite desejável de 200ms. | Verificado nas execuções locais de testes de integração. |
| **RNF011** | Documentação (Swagger) | **Atendido.** Geração automática e interativa de documentação OpenAPI 3.0 via Swagger UI. | Dependência `springdoc-openapi` no [pom.xml](file:///c:/Users/ronye/Documents/sgmc/pom.xml#L73) e interfaces OpenAPI como [MembroControllerOpenAPI.java](file:///c:/Users/ronye/Documents/sgmc/src/main/java/br/com/mam/sgmc/api/openapi/MembroControllerOpenAPI.java). |
